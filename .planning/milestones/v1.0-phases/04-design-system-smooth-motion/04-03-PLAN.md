## Phase Goal

**As an** investor, **I want to** switch between the global map and company profiles with smooth, mental-map-preserving motion, **so that** the visualization never disorients me with a jarring layout reset.

<objective>
Split the viz render lifecycle into a one-time buildSimulation() and an idempotent updateGraph(graph), so view changes re-bind data and gently reheat instead of tearing down and recreating the force simulation. Carry node positions across switches and gate D3 transitions behind a prefers-reduced-motion guard.

Purpose: this is the core STORY-03 fix. Today render() calls clearGraph() (stop + remove everything) then d3.forceSimulation(...) fresh each call (L411/L589), so nodes fly from random seeds on every mode switch (openProfile, openGlobal, loadView). Building once and re-binding with simulation.nodes()/force("link").links() + alphaTarget(0.3) reheat preserves the mental map.

Output: refactored js/viz/index.js with buildSimulation + updateGraph; render() kept as the public entry so the three ui call sites and main.js stay unchanged. Implements STORY-03.
</objective>
